1. What is CI/CD?
Answer:
CI/CD stands for Continuous Integration and Continuous Delivery/Deployment.

CI (Continuous Integration): Developers frequently merge code into a shared repository, and automated builds/tests run to detect issues early.
CD (Continuous Delivery/Deployment): Automates the release of applications to environments (staging/production).


✅ 2. What is the difference between Continuous Delivery and Continuous Deployment?
Answer:

Continuous Delivery: Code changes are automatically tested and prepared for release, but deployment to production requires manual approval.
Continuous Deployment: Every change that passes tests is automatically deployed to production without manual intervention.


✅ 3. What are the benefits of CI/CD?
Answer:

Faster release cycles
Early bug detection
Improved code quality
Reduced manual effort
More reliable deployments


✅ 4. What tools are commonly used in CI/CD?
Answer:
Some popular tools include:

CI Tools: Jenkins, GitHub Actions, GitLab CI, CircleCI
Build Tools: Maven, Gradle
Containerization: Docker
Orchestration: Kubernetes
Deployment: Ansible, Terraform


✅ 5. What is a CI/CD pipeline?
Answer:
A CI/CD pipeline is an automated workflow that includes steps like:

Code commit
Build
Test
Deploy

It ensures code moves from development to production efficiently and reliably.

✅ 6. What is version control and why is it important in CI/CD?
Answer:
Version control (e.g., Git) tracks changes in code and allows multiple developers to collaborate.
It is important because CI/CD pipelines are triggered by code changes in repositories.

✅ 7. What is a build in CI/CD?
Answer:
A build is the process of compiling source code into an executable artifact (like a JAR, WAR, or Docker image).
It ensures that the code is syntactically correct and ready for testing.

✅ 8. What is automated testing in CI/CD?
Answer:
Automated testing involves running tests (unit, integration, etc.) automatically during the pipeline to verify code quality and functionality.

✅ 9. What is rollback in CI/CD?
Answer:
Rollback is the process of reverting to a previous stable version if a deployment fails or causes issues in production.

✅ 10. What is Infrastructure as Code (IaC)?
Answer:
IaC is the practice of managing and provisioning infrastructure using code instead of manual processes.
Tools include:

Terraform
AWS CloudFormation

It helps in consistency, scalability, and automation.
